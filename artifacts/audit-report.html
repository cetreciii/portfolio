2
Infinite loading skeleton on network error
components/Articles.tsx · line 32
Bug High ✓ Fixed
What was wrong

The getMediumArticles call in Articles.tsx had no .catch(). If the rss2json API was unreachable (API down, no internet), the async function would throw, .then() would never fire, and setLoading(false) would never be called — leaving visitors staring at animated skeleton cards with no way out. About.tsx already had a .catch(() => {}) on the same call; Articles was just missing it.

Fix applied
components/Articles.tsx
getMediumArticles("igor.tarantino").then((data) => {
setArticles(data);
setLoading(false);
});
getMediumArticles("igor.tarantino")
.then((data) => {
setArticles(data);
setLoading(false);
})
.catch(() => setLoading(false));

On error, articles stays [] and loading becomes false, which triggers the existing if (!loading && articles.length === 0) return null guard — the section hides cleanly.
